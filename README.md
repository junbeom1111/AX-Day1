# Markdown 보고서

## 1. 서론

현대의 디지털 문서 작성 환경에서는 **간결하면서도 구조화된 표현 방식**이 중요해지고 있다.  
이러한 흐름 속에서 등장한 **Markdown**은 일반 텍스트 기반으로 문서를 작성하면서도, 문서의 구조와 의미를 쉽게 표현할 수 있도록 도와주는 경량 마크업 언어이다.

Markdown은 복잡한 태그를 직접 입력해야 하는 HTML보다 훨씬 간단한 문법을 사용하며, 문서를 **읽기 쉽게 작성할 수 있다는 점**에서 큰 장점을 가진다. 특히 개발 문서, README 파일, 블로그 글, 노트 정리 등 다양한 분야에서 널리 활용되고 있다.

> Markdown의 가장 큰 특징은  
> **쓰기 쉽고, 읽기 쉽다**는 점이다.

본 보고서에서는 Markdown의 개념과 특징, 설계 철학, 주요 문법, 활용 사례, 장단점 등을 종합적으로 살펴보고자 한다.

---

## 2. Markdown의 개념과 특징

### 2.1 Markdown의 정의

Markdown은 **일반 텍스트(plain text)** 형식으로 문서를 작성하면서도, 이를 HTML과 같은 형식으로 쉽게 변환할 수 있도록 고안된 문법 체계이다.

쉽게 말하면, 다음과 같이 정리할 수 있다.

- 사람이 읽기 쉬운 텍스트 문서를 작성할 수 있다.
- 일정한 문법을 사용하여 문서 구조를 표현할 수 있다.
- HTML로 변환하여 웹 문서로 활용할 수 있다.

### 2.2 Markdown의 핵심 특징

Markdown의 주요 특징은 다음과 같다.

1. 문법이 단순하다.
2. 원본 텍스트 자체의 가독성이 높다.
3. 다양한 플랫폼과 도구에서 지원된다.
4. HTML로 변환이 쉽다.
5. 협업과 버전 관리에 유리하다.

### 2.3 HTML과의 차이

다음은 HTML과 Markdown의 차이를 간단히 비교한 예시이다.

| 구분 | HTML | Markdown |
|---|---|---|
| 작성 방식 | 태그 중심 | 기호 중심 |
| 가독성 | 상대적으로 낮음 | 높음 |
| 학습 난이도 | 다소 높음 | 비교적 낮음 |
| 활용 목적 | 정교한 웹 표현 | 간단한 문서 작성 |

예를 들어, HTML에서는 제목을 다음과 같이 작성한다.

    <h1>제목</h1>

반면 Markdown에서는 훨씬 간단하게 작성할 수 있다.

    # 제목

이처럼 Markdown은 최소한의 문법으로 문서의 의미를 표현할 수 있다는 장점이 있다.

## 3. Markdown의 설계 철학

Markdown은 단순한 문법 모음이 아니라, 분명한 설계 철학을 바탕으로 만들어졌다.

### 3.1 읽기 쉬운 문서

Markdown은 변환 이전의 원본 텍스트만 보더라도 문서 구조를 쉽게 이해할 수 있도록 설계되었다. 즉, 문서 작성자가 HTML로 변환하지 않아도 내용을 읽고 파악할 수 있어야 한다는 생각이 반영되어 있다.

예를 들어, 아래 문장을 보자.

> 이것은 인용문이다.

이 문법은 실제로도 인용문처럼 보이기 때문에, 원본 자체만 봐도 의미를 추측할 수 있다.

### 3.2 쓰기 쉬운 문서

Markdown은 자주 사용하는 서식을 빠르게 입력할 수 있도록 구성되어 있다. 복잡한 태그를 여닫을 필요 없이, 간단한 기호 몇 개만으로 구조를 표현할 수 있다.

예시:

- `#` : 제목
- `*` 또는 `-` : 목록
- `**텍스트**` : 굵게
- `` `코드` `` : 인라인 코드

### 3.3 최소한의 문법

Markdown은 문법이 지나치게 복잡해지는 것을 지양한다. 복잡한 레이아웃이나 세부 스타일 지정은 HTML에 맡기고, Markdown은 **핵심적인 문서 구조 표현**에 집중한다.

## 4. Markdown의 주요 문법

이 장에서는 Markdown에서 자주 사용되는 대표 문법을 살펴본다.

### 4.1 제목(Headers)

제목은 `#` 기호를 사용하여 표현한다. `#`의 개수에 따라 제목의 수준이 달라진다.

    # 제목 1
    ## 제목 2
    ### 제목 3

실제 예시는 다음과 같다.

# 제목 1 예시
## 제목 2 예시
### 제목 3 예시

---

### 4.2 문단과 줄바꿈

Markdown에서는 일반적으로 **한 줄 이상 비우면 새로운 문단**이 된다.

    첫 번째 문단입니다.

    두 번째 문단입니다.

줄바꿈은 환경에 따라 다르게 처리될 수 있으므로, 문단 구분은 빈 줄을 사용하는 것이 가장 명확하다.

---

### 4.3 강조(Emphasis)

강조는 다음과 같이 표현할 수 있다.

- *기울임*
- **굵게**
- ***굵고 기울임***
- ~~취소선~~

문법 예시는 다음과 같다.

    *기울임*
    **굵게**
    ***굵고 기울임***
    ~~취소선~~
문서에서 중요한 개념을 설명할 때 이러한 강조 문법은 매우 유용하다. 예를 들어, **Markdown의 핵심 가치는 가독성**이라고 할 수 있다.

---

### 4.4 목록(Lists)

#### 순서 없는 목록

    - 항목 1
    - 항목 2
      - 하위 항목 2-1
      - 하위 항목 2-2

예시:

- 항목 1
- 항목 2
  - 하위 항목 2-1
  - 하위 항목 2-2

#### 순서 있는 목록

    1. 첫 번째
    1. 두 번째
    1. 세 번째

예시:

1. 첫 번째
1. 두 번째
1. 세 번째

순서 있는 목록은 일반적으로 `1.`, `2.`, `3.`처럼 차례대로 작성할 수도 있고,  
`1.`, `1.`, `1.`처럼 모두 같은 숫자로 입력해도 많은 Markdown 환경에서는 자동으로 번호가 순서대로 표시된다.  
이 방식은 항목을 중간에 추가하거나 순서를 수정할 때 번호를 일일이 다시 고치지 않아도 된다는 장점이 있다.  
다만 일부 환경에서는 작성한 숫자 그대로 보이거나 렌더링 방식이 다를 수 있으므로, 문서의 일관성과 가독성을 위해 필요한 경우 `1, 2, 3`처럼 직접 맞춰 쓰는 방법도 자주 사용된다.

목록 문법은 단계 설명이나 특징 정리에 특히 효과적이다.

---

### 4.5 인용문(Blockquotes)

인용문은 `>` 기호를 사용한다.

    > Markdown은 읽기 쉬운 평문 형식을 지향한다.

예시:

> Markdown은 읽기 쉬운 평문 형식을 지향한다.

중첩 인용도 가능하다.

    > 첫 번째 인용
    >> 두 번째 인용

예시:

> 첫 번째 인용
>> 두 번째 인용

---

### 4.6 코드(Code)

Markdown은 코드 표현에 매우 유용하다.

#### 인라인 코드

문장 중간의 코드는 백틱을 사용한다.

입력 예시:

    `print("Hello, Markdown")`

결과 예시:

`print("Hello, Markdown")`

#### 코드 블록

여러 줄의 코드는 백틱 3개(` ``` `) 또는 물결표 3개(`~~~`)로 감싸서 작성한다.  
또한 시작 부분에 언어 이름을 함께 적으면, 일부 Markdown 환경에서는 문법 강조(syntax highlighting)도 적용할 수 있다.

입력 예시:

    ~~~python
    def hello():
        print("Hello, Markdown")

    hello()
    ~~~

결과 예시:

~~~python
def hello():
    print("Hello, Markdown")

hello()
~~~

이처럼 Markdown은 짧은 인라인 코드뿐 아니라 여러 줄의 코드 블록도 쉽게 표현할 수 있기 때문에, 프로그래밍 문서나 기술 보고서 작성에서 매우 유용하다.

---
### 4.7 링크(Links)

링크는 다음과 같이 작성한다.

    [대체 텍스트](링크주소 "링크에 대한 설명")
    [DARING FIREBALL](https://daringfireball.net/projects/markdown "마크다운에 대해 정리한 웹사이트")

예시: [DARING FIREBALL By John Gruber](https://daringfireball.net/projects/markdown "마크다운에 대해 정리한 웹사이트")

문서 작성 시 참고 자료나 관련 사이트를 연결하는 데 유용하다.

---

### 4.8 이미지(Images)

이미지는 링크 문법 앞에 `!`를 붙여 작성한다.

    ![대체 텍스트](이미지주소)
    ![Markdown 로고](https://upload.wikimedia.org/wikipedia/commons/thumb/4/48/Markdown-mark.svg/250px-Markdown-mark.svg.png)
예시: 

![Markdown 로고](https://upload.wikimedia.org/wikipedia/commons/thumb/4/48/Markdown-mark.svg/250px-Markdown-mark.svg.png)

실제 제출용 보고서에서는 로컬 이미지나 스크린샷을 첨부하는 방식으로 활용할 수 있다.

---

### 4.9 수평선(Horizontal Rules)

내용 구분을 위해 수평선을 사용할 수 있다.

    ---

또는

    ***

예시:

---

이 문법은 문서의 장과 절을 시각적으로 구분하는 데 효과적이다.

## 5. Markdown의 확장 요소

Markdown은 기본 문법 외에도 여러 부가 기능을 지원한다.

### 5.1 체크리스트

일부 Markdown 환경에서는 체크리스트를 지원한다.

    - [x] Markdown 개념 정리
    - [x] 기본 문법 조사
    - [ ] 실습 예시 추가

예시:

- [x] Markdown 개념 정리
- [x] 기본 문법 조사
- [ ] 실습 예시 추가

### 5.2 표(Table)

표는 정보를 구조적으로 정리할 때 유용하다.

    | 문법 | 기능 |
    |---|---|
    | `#` | 제목 |
    | `-` | 목록 |
    | `>` | 인용문 |
    | `[]()` | 링크 |

예시:

| 문법 | 기능 |
|---|---|
| `#` | 제목 |
| `-` | 목록 |
| `>` | 인용문 |
| `[]()` | 링크 |

### 5.3 HTML 혼합 사용

Markdown에서는 필요한 경우 HTML을 함께 사용할 수 있다.

    <b>굵은 글씨</b>
    <br>
    <i>기울인 글씨</i>

즉, Markdown으로 표현하기 어려운 부분은 HTML을 보완적으로 사용할 수 있다.

## 6. 예시 자료를 통한 Markdown 이해

Markdown은 문법 정의만으로 배우는 것보다, **실제 화면과 함께 볼 때 훨씬 직관적으로 이해**할 수 있다.  
특히 실시간 편집기, GitHub README, 치트시트, 입문 영상 등은 Markdown의 활용성과 직관성을 효과적으로 보여준다.

### 6.1 실시간 미리보기 화면

이 방식은 Markdown의 가장 큰 장점인 **원문과 결과를 동시에 비교할 수 있음**을 잘 보여준다.

## 7. Markdown의 활용 사례

Markdown은 여러 분야에서 폭넓게 사용된다.

### 7.1 개발 문서

개발자들은 프로젝트 설명 파일인 `README.md`를 작성할 때 Markdown을 자주 사용한다.

예를 들어 README에는 다음과 같은 내용이 들어갈 수 있다.

- 프로젝트 소개
- 설치 방법
- 실행 방법
- 예제 코드
- 라이선스 정보

### 7.2 블로그 작성

일부 정적 사이트 생성기나 블로그 플랫폼은 Markdown 문서를 직접 지원한다. 따라서 작성자는 복잡한 편집기 없이도 깔끔한 글을 작성할 수 있다.

### 7.3 학습 및 노트 정리

학생이나 연구자는 Markdown을 활용해 강의 노트, 실험 기록, 보고서 초안 등을 정리할 수 있다.

예를 들면 다음과 같은 형식이 가능하다.

    ## 오늘 학습한 내용
    - Markdown 제목 문법
    - 코드 블록 작성법
    - 표 만들기

### 7.4 협업 문서

Markdown은 Git과 같은 버전 관리 시스템과 잘 어울린다. 텍스트 기반 문서이기 때문에 변경 이력을 추적하기 쉽고, 협업 과정에서도 효율적이다.

## 8. Markdown의 장점과 한계

### 8.1 장점

Markdown의 대표적인 장점은 다음과 같다.

- 문법이 간단하여 배우기 쉽다.
- 원본 문서의 가독성이 높다.
- 다양한 에디터와 플랫폼에서 지원된다.
- HTML 변환이 가능하다.
- 협업 및 버전 관리에 적합하다.

특히 초보자도 비교적 짧은 시간 안에 익힐 수 있다는 점은 매우 큰 장점이다.

### 8.2 한계

반면 Markdown에는 다음과 같은 한계도 존재한다.

- 표준이 완전히 통일되어 있지 않다.
- 도구마다 지원 문법이 조금씩 다르다.
- 복잡한 레이아웃 표현에는 한계가 있다.
- 세부적인 디자인 제어는 어렵다.

예를 들어, 어떤 환경에서는 체크리스트가 잘 동작하지만 다른 환경에서는 단순 텍스트로만 보일 수 있다.

### 8.3 종합적 평가

Markdown은 모든 문서를 대체하는 만능 도구는 아니지만, **간결하고 구조적인 텍스트 문서 작성**에는 매우 뛰어난 도구라고 평가할 수 있다.

## 9. 실습 예시

다음은 Markdown으로 간단한 학습 메모를 작성한 예시이다.

    # Python 학습 정리

    ## 1. 변수
    - 정수형
    - 문자열
    - 실수형

    ## 2. 출력
        print("Hello")

    ## 3. 느낀 점
    > 문법이 직관적이어서 정리하기 쉽다.

위와 같은 형식은 수업 내용 정리, 프로젝트 기록, 개인 노트 작성 등에 모두 활용될 수 있다.

---

## 10. 결론

Markdown은 단순한 문법 체계를 넘어, **읽기 쉽고 쓰기 쉬운 문서 작성 방식**이라는 점에서 큰 의미를 가진다. 복잡한 편집 도구 없이도 구조화된 문서를 작성할 수 있으며, HTML 변환도 가능하다는 점에서 실용성이 매우 높다.

또한 Markdown은 개발, 교육, 협업, 블로그 작성 등 다양한 환경에서 널리 사용되며, 디지털 시대의 효율적인 문서 작성 도구로 자리 잡고 있다.

정리하자면 Markdown은 다음과 같은 가치를 가진다.

- 간결성
- 가독성
- 확장성
- 실용성

따라서 Markdown은 앞으로도 다양한 분야에서 지속적으로 활용될 가능성이 높다.

## 부록. 자주 쓰는 Markdown 문법 요약

| 표현 요소 | 문법 예시 |
|---|---|
| 제목 | `# 제목` |
| 소제목 | `## 소제목` |
| 굵게 | `**굵게**` |
| 기울임 | `*기울임*` |
| 목록 | `- 항목` |
| 번호 목록 | `1. 항목` |
| 인용문 | `> 인용문` |
| 링크 | `[이름](주소)` |
| 이미지 | `![설명](주소)` |
| 인라인 코드 | `` `code` `` |
| 코드 블록 | 백틱 3개 또는 물결표 3개 사용 |
